查看git提交日志: git log --pretty=oneline
回到某个版本 git reset --hard 1094a(回到未来的某个版本,其中1094a是未来的某个commit id,也就是hard参数后面可以跟版本号)
			git reset --hard 1094a使用这个命令后,不管之前是什么状态的,都将是的工作区 暂存区 版本库都是干净的.

用来记录你的每一次命令 git reflog
查看某个文件的改动 git diff 文件名 (与最新的跟踪点进行比对,比如最新是提交到了版本库，就与版本库做比较,如果是添加到了暂存区，就与暂存区做比较)
git commit就可以一次性把暂存区的所有修改提交到分支
git diff HEAD -- 文件名 命令可以查看工作区和版本库里面最新版本的区别

命令git checkout -- readme.txt意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：
   一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
   一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态

git reset HEAD <file>可以把暂存区的修改撤销掉（unstage）,工作区的内容并不会发生改变(并不是让工作区的文件内容跟版本库中的一模一样)

冲突 (相当于你是删除了一行 又新增了一行)



删除文件，步骤 rm 文件名----> git rm 文件名 ----> git commit -m 'delete file'(commit会一次性把暂存区的所有修改提交到分支)
